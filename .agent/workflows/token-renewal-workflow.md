# Token Renewal Workflow

Manages rotation of expiring CI/CD tokens to prevent publish failures.

## Token Inventory

| Secret | Platform | Expiry Model | Workflow File | Current Expiry |
|--------|----------|-------------|---------------|----------------|
| `NPM_TOKEN` | npmjs.com | 90-day granular | `publish.yml` | **2026-08-29** |
| `GITHUB_TOKEN` | GitHub | Auto (per-run) | `release.yml` | N/A (automatic) |

> [!NOTE]
> `GITHUB_TOKEN` in workflows is auto-generated by GitHub Actions per-run. No rotation needed.
> The local `GITHUB_TOKEN` env var injected by Antigravity IDE is a dummy placeholder — also no rotation needed.

---

## When to Renew

| Signal | Action |
|--------|--------|
| 7 days before expiry | Proactive renewal (recommended) |
| `npm ERR! 404 Not Found` on PUT | Token expired — immediate renewal |
| `npm ERR! 401 Unauthorized` | Token revoked or expired |
| `npm ERR! 403 Forbidden` | Token scoped incorrectly |
| Release workflow fails at `publish-npm` | Check token first |

## Renewal Steps

### Step 1: Generate New npm Token

1. Go to: **https://www.npmjs.com/settings/matbanik/tokens/granular-access-tokens/new**
2. Fill in:

| Field | Value |
|-------|-------|
| Token name | `github-actions-publish` |
| Description | `CI/CD publish token for pomera-ai-commander` |
| Bypass 2FA | ☑ Checked |
| Packages → Permissions | **Read and write** |
| Select packages | `pomera-ai-commander` |
| Organizations | No access |
| Expiration | **90 days** |

3. Click **Generate token**
4. Copy the token immediately

### Step 2: Update GitHub Secret

**Option A — CLI (preferred for agents):**

```powershell
# Clear Antigravity's dummy token first
$env:GITHUB_TOKEN=''

# Set the new token (pipe or interactive)
echo "npm_XXXXX..." | gh secret set NPM_TOKEN

# Verify
gh secret list
```

**Option B — Web UI:**

1. Go to: https://github.com/matbanik/Pomera-AI-Commander/settings/secrets/actions
2. Click pencil icon on `NPM_TOKEN`
3. Paste new token → Save

### Step 3: Delete Old Token

1. Go to: https://www.npmjs.com/settings/matbanik/tokens
2. Find the old `github-actions-publish` token (check expiry date)
3. Delete it

### Step 4: Verify

**If a release just failed:**

```powershell
# Re-run only the failed jobs
$env:GITHUB_TOKEN=''
gh run list --limit 3
gh run rerun <RUN_ID> --failed
```

**For proactive renewal (no failed release):**

```powershell
# Quick validation — trigger a no-op version check
$env:GITHUB_TOKEN=''
gh workflow run publish.yml --field publish_pypi=false --field publish_npm=true
```

### Step 5: Update Expiry Records

Update **two places**:

1. **Token History** in `.agent/docs/npm-token-github-guide.md`:
   ```markdown
   | 2026-MM-DD | **2026-MM-DD** | Rotated; 90-day granular token |
   ```

2. **This workflow** — update the `Current Expiry` in the Token Inventory table above.

3. **Save to Pomera notes** for continuity:
   ```
   pomera_notes action=save
     title="Token/YYYY-MM-DD/npm-token-rotated"
     input_content="NPM_TOKEN rotated for pomera-ai-commander"
     output_content="New expiry: YYYY-MM-DD. Updated npm-token-github-guide.md and token-renewal-workflow.md"
   ```

## Agentic Automation

When an AI agent handles token renewal, it should:

```
1. User provides new token value
2. Clear GITHUB_TOKEN:  $env:GITHUB_TOKEN=''
3. Set secret:          echo "<token>" | gh secret set NPM_TOKEN
4. Verify:              gh secret list
5. Re-run if needed:    gh run rerun <ID> --failed
6. Update docs:         npm-token-github-guide.md (history table)
7. Update workflow:     token-renewal-workflow.md (Current Expiry)
8. Save note:           pomera_notes (Token/ category)
```

### Important: GITHUB_TOKEN Override

Antigravity IDE injects a **dummy** `GITHUB_TOKEN` process env var (`github_pat_antigravitydummytoken`). This overrides `gh` CLI's keyring auth.

**Always clear it before `gh` commands:**
```powershell
$env:GITHUB_TOKEN=''
```

This is process-scoped — it doesn't affect other terminals or persist.

## Troubleshooting

### npm E404 on PUT (most common)

```
npm error code E404
npm error 404 Not Found - PUT https://registry.npmjs.org/pomera-ai-commander
```

**Cause:** Token expired. npm returns 404 (not 401) for expired granular tokens on PUT.
**Fix:** Follow renewal steps above.

### gh CLI returns 401

```
non-200 OK status code: 401 Unauthorized body: "Bad credentials"
```

**Cause:** Antigravity IDE's dummy `GITHUB_TOKEN` env var is overriding keyring auth.
**Fix:** `$env:GITHUB_TOKEN=''` before any `gh` command.

### PyPI publish fails

PyPI uses **trusted publishing** (OIDC) — no token needed. If it fails:
1. Check the GitHub Environment `pypi` is configured
2. Verify at: https://pypi.org/manage/project/pomera-ai-commander/settings/publishing/
